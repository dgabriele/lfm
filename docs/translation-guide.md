# IPA -> English Translation Guide

Self-supervised translation from emergent IPA utterances back to English, closing the interpretability loop: agent -> LFM -> IPA -> LLM -> English.

## Overview

LFM agents produce emergent IPA utterances through the frozen decoder. The translator module maps these back to English by:

1. **Generating parallel pairs**: English sentences are encoded via sentence-transformer, passed through the frozen faculty to produce IPA, creating (IPA, English) training data.
2. **Fine-tuning a causal LM**: A small language model learns the IPA -> English mapping using the format `<ipa> ... </ipa> <eng> ... </eng>`.
3. **Evaluating**: BLEU scores and semantic cosine similarity measure translation quality.

This is "translation, not alignment" — the agent's ontology stays intact; the LLM does the interpretive work.

## Quick Start

```bash
# Install translator dependencies
poetry install --with translator

# Step 1: Generate cached (IPA, English) pairs
lfm translate generate-pairs \
  --decoder-path data/models/v1/vae_decoder.pt \
  --spm-path data/models/v1/spm.model \
  --max-sentences 5000 \
  --output data/models/v1/translator/pairs.jsonl

# Step 2: Train translator
lfm translate train \
  --model-name Qwen/Qwen2.5-0.5B \
  --pairs data/models/v1/translator/pairs.jsonl \
  --epochs 3

# Step 3: Evaluate
lfm translate eval \
  --model-dir data/models/v1/translator

# Step 4: Visualize results
lfm visualize translation \
  --results-dir data/models/v1/translator
```

## Architecture

### Why Separate Steps?

Pair generation and training are separate CLI commands because:

- **VRAM**: Pair generation loads sentence-transformer + faculty (~2GB), training loads the LLM (~1-3GB). Can't coexist on 8GB GPU.
- **Caching**: Pairs on disk (JSONL) are reusable across training experiments with different hyperparams or LLMs.
- **Debuggability**: Pairs can be inspected independently.

### VRAM Budget (8GB GPU)

| Stage | Components | Est. VRAM |
|-------|-----------|-----------|
| Pair gen: encode | sentence-transformer (384-dim) | ~0.5GB |
| Pair gen: faculty | frozen decoder (27M params fp32) | ~1.5GB |
| Training | Qwen 0.5B fp16 + optimizer states | ~3GB |
| Training + LoRA | Qwen 1.5B fp16 + LoRA adapters | ~5GB |
| Eval | model fp16 + generation cache | ~2GB |

All stages are sequential — never more than one heavy model loaded.

### Dataset Format

Each training example has the format:

```
<ipa> {ipa_text} </ipa> <eng> {english_text} </eng>
```

Labels mask the prompt portion (`<ipa>...</ipa> <eng>`) with -100, so the model only trains on generating the English output.

## Configuration

Default config is at `configs/translate.yaml`. CLI flags override config defaults.

### Pair Generation

```python
PairGenerationConfig(
    leipzig_dir="data/leipzig",
    languages=["eng"],
    max_sentences=5000,
    decoder_path="data/models/v1/vae_decoder.pt",
    spm_path="data/models/v1/spm.model",
    encoder_model="all-MiniLM-L6-v2",
    encode_batch_size=64,
    output_path="data/models/v1/translator/pairs.jsonl",
)
```

### Translator Training

```python
TranslatorConfig(
    model_name="Qwen/Qwen2.5-0.5B",
    use_lora=False,          # Enable for larger models
    epochs=3,
    lr=2e-5,
    batch_size=8,
    gradient_accumulation_steps=4,
    use_amp=True,
    warmup_fraction=0.1,
)
```

### Using LoRA

For larger models (1.5B+) that don't fit in VRAM for full fine-tuning:

```bash
lfm translate train \
  --model-name Qwen/Qwen2.5-1.5B \
  --use-lora \
  --batch-size 4
```

## Artifacts

After a full pipeline run, the output directory contains:

```
data/models/v1/translator/
  pairs.jsonl                # Cached (IPA, English) pairs
  pairs_config.yaml          # PairGenerationConfig snapshot
  model/                     # HuggingFace save_pretrained output
    config.json
    model.safetensors
    tokenizer.json
    ...
  config.yaml                # TranslatorConfig snapshot
  training_history.json      # Session history
  results.json               # BLEU + semantic similarity metrics
  translations.jsonl         # Per-example translations with similarity
```

## Evaluation Metrics

- **BLEU-1/2/3/4**: Corpus-level n-gram precision with brevity penalty
- **BLEU-4 (geometric)**: Standard BLEU-4 geometric mean
- **Semantic cosine similarity**: Sentence-transformer re-embedding of reference vs hypothesis

## Visualizations

Three figures generated by `lfm visualize translation`:

1. **Similarity histogram**: Per-example cosine similarity distribution with mean/median lines
2. **BLEU breakdown**: Grouped bar chart of BLEU-1/2/3/4 + geometric mean
3. **Examples table**: Top 10 translations color-coded by similarity (green/yellow/red)
